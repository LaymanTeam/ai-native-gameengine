Synthesized pixel art library and curated open-source asset repositories
Pixel art (node): pixel-sprite-generator (zfedoran). It's the canonical procedural sprite generator — sprites are algorithmically generated by combinatorial methods using a 2D mask whose values are randomized and mirrored. The core lib renders to a canvas, so for headless node→PNG you pair it with node-canvas. There's a ready-made wrapper, pixel-sprite-generator-nodejs, that creates procedural 2D creatures as .png files directly from node — but flag it: it's an inactive project with very low usage and a Snyk scan reporting three vulnerabilities/license issues, last updated in 2023. So either use the wrapper knowing that, or run the core algorithm yourself with node-canvas (and there's a TypeScript rewrite by seiyria if you want types). For pattern-defined sprites rather than fully random, pixel-art-gen is the alternative.

Two asset domains (music + sfx + art, searchable):

OpenGameArt.org — one repository for all three. It hosts 2D/3D art, sound effects, and music, all under free licenses, and rejects NC/ND clauses so everything is cleared for commercial reuse and remixing. Searchable URL: https://opengameart.org/art-search-advanced?keys=YOUR_QUERY (filterable by art type — 2D, music, sound effect).
Wikipedia
Kenney.nl — also covers all three, and most assets are CC0 (public domain), usable and modifiable without attribution, with a clean consistent style. Browse/search at https://kenney.nl/assets.
